# 架构总结：Pretext 引力场演化项目

本文档是对 `pretext2/index.html` 单文件应用的系统架构总结。本项目展示了如何将纯文本渲染转化为一套全动态的高级物理粒子系统，并配合现代网页前沿的交互设计。

## 1. 核心模块与入口机制
本应用基于 **Vanilla Canvas 2D API**，未使用任何沉重的 DOM 框架（如 React/Vue）。
- **`importmap` (ESM 模块导入)**：在 `<head>` 中使用了原生 Import Maps 机制直接从 CDN (`esm.sh`) 加载 `@chenglou/pretext` (排版引擎) 和 `lil-gui` (参数控制台)。
- **入口函数 (`initSystem`)**：项目加载完成后通过 `initSystem()` 测算初始宽位、排版引擎对文字进行预处理与断行，并最终将所有生成的文字切割成微观的物理粒子对象实体。
- **渲染循环 (`loop`)**：通过 `requestAnimationFrame` 无限驱动高帧率引擎，在这里更新时间差(dt)、清空画布、并依次触发各单位的物理计算与视觉渲染。

## 2. 界面与样式层 (UI & UX)
- **CSS 动态主题体系**：通过在 `<body>` 根节点切换 `data-theme` 属性（`light` / `dark`），CSS 会通过内置延时的 `transition` 和贝塞尔曲线实现底层背景颜色、字体颜色和组件配色的丝滑反转。并以此对应物理机制中的**白洞（发射/黑夜）**与**黑洞（吞噬/白天）**概念。
- **高级定制化 Toggle 开关**：舍弃了浏览器原生的 Checkbox 样式，纯 CSS 编写了一个具有 iOS 果冻过渡质感的胶囊开关。
- **玻璃态 `lil-gui`**：使用大量局部 CSS 变量对第三方组件 `lil-gui` 进行深度覆写，实现了带有 `backdrop-filter: blur` 的高级玻璃态设计，并且该面板随昼夜主题自动响应高对比度配色。

## 3. 物理与实体模型层 (Entities)

引擎中包含了几个关键的角色对象：

### ① 游荡的系统轴心 (System Center)
包含了 `system.x` 和 `system.y` 两大中心坐标。它是白洞或黑洞实际渲染的根据地。
- **逻辑**：通过基于 `mousemove / touchmove` 的缓动插值（Lerp），实现带惯性的随带光标追随。

### ② 发光体：黑洞 (Black Hole) / 白洞 (White Hole)
- **视觉**：不通过外部贴图，而是高度依赖 Canvas 原生的 `createRadialGradient` 动态生成深邃或耀眼的径向星云光环，再伴随 `glow / shadowBlur` 模拟内核的高亮或深邃。
- **形态切换**：通过一个缓动变量 `scale`（0 到 1），让昼夜切换时两个形体自然缩放显隐。同时含有视觉脉冲变量 `pulse`，在开枪喷吐时能引发星体收缩膨胀。

### ③ 全动态粒子类 (Particle)
代表排版中的每一个*独立中文字符*，是整个项目的核心计算密集区。
该类拥有内部状态机：`state = 'idle' | 'swallowed' | 'knocked_down'`
- **IDLE 状态 (悬浮态)**：
  - 受弹簧力（`springStiffness`）持续向自身的原始排版坐标 `originX/Y` 拉扯。
  - 会发生物理碰撞：画布上下左右四面边缘约束。
  - 受外界引力影响：根据距离公式计算平方成反比的引力/斥力（受白洞反弹，受黑洞牵引）。
- **SWALLOWED 状态 (吞噬态)**：
  - 彻底从渲染管线中被剔除，存入黑洞的空间向量数组 `swallowedParticles` 里，待被吐出。
- **KNOCKED_DOWN 状态 (坠落堆叠态)**：
  - 被白洞子弹命中后进入该状态。失去被弹簧拉回原来排版的记忆，完全服从于宏观世界的 1.0G 物理重力加速度（`vy += 1.0`）。
  - **基于位置约束的动力学 (Position Based Dynamics, PBD)**：落在底部时强制剔除纵向反弹动能，文字与文字重叠时只进行生硬的向上抬升。彻底解决了海量刚体互相过度修正而导致的鬼畜抖动，实现如沙般堆积。

### ④ 动能光弹类 (Bullet)
由白洞点击发射（或长按连射）。具有高速动能、碰撞判断等物理特性，遇到处于 `'idle'` 的汉字会击碎它，遇到处于 `'knocked_down'` 的汉字会将其动能补满，重新打回半空，从而触发使其被原位弹簧再次捕获（创世复苏机制）。

## 4. 特殊技术与性能调优
- **`requestAnimationFrame` 无限清理策略**：背景采用了 `clearRect` 而不是常见的实色矩形覆盖，这是刻意为之以让 Canvas 完全透明，透出 HTML 底层的 CSS 平滑变色渐变背景，节约了重绘开销的同时加强了质感。
- **交互不重绘 (`resize` 防抖重绑定)**：在监听到屏幕拉伸缩放时，不会去无脑摧毁和重新 `new Particle`！系统会安静地重新对齐排版要求，仅仅只是把新计算的位置数据热更新给了 `originX` 和 `originY`。这使得粒子能在任意拖拽下，始终具有流畅拉扯碰撞的平滑视觉记忆体验。
